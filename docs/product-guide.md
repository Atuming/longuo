# 火龙果编辑器 — 产品说明书

## 产品简介

火龙果编辑器是一款专为中文小说作者打造的 Web 端写作工具。它将写作编辑器、AI 辅助创作、角色管理、世界观构建、时间线追踪和多格式导出整合在一个简洁的界面中，帮助作者专注于创作本身。

所有数据存储在你的本地电脑上，无需注册账号，无需联网（AI 功能除外），打开浏览器即可开始写作。

---

## 快速开始

### 安装与启动

```bash
# 安装依赖
npm install

# 启动开发服务器
npm run dev
```

浏览器打开 `http://localhost:5173` 即可使用。Windows 用户也可以双击 `启动火龙果编辑器.bat` 一键启动。

### 创建第一个项目

1. 在首页点击「新建项目」
2. 输入小说名称和简介
3. 选择保存位置，生成 `.novel` 文件
4. 进入编辑器，开始创作

## 功能详解

### 一、写作编辑器

火龙果编辑器的核心是一个基于 CodeMirror 6 的 Markdown 编辑器。

- **Markdown 语法高亮**：标题、粗体、斜体、引用、列表等 Markdown 语法实时渲染
- **角色名高亮**：你定义的角色名称会在编辑器中自动高亮显示，一眼就能看到哪些角色出现在当前章节
- **@提及补全**：输入 `@` 后自动弹出角色列表，选择即可插入角色名
- **悬浮信息卡**：鼠标悬停在角色名上，显示角色的外貌、性格等关键信息
- **专注模式**：点击工具栏的专注模式按钮，隐藏侧边栏和面板，只留编辑器和精简工具栏
- **日更目标**：设置每日写作字数目标，编辑器底部实时显示今日进度

### 二、章节管理

左侧大纲面板以树形结构展示你的小说目录。

- **三级层级**：卷 → 章 → 节，适合长篇小说的结构化管理
- **拖拽排序**：直接拖拽章节调整顺序，也可以拖到其他卷下改变层级
- **字数统计**：每个章节旁显示字数，自动去除 Markdown 标记后计算
- **快速操作**：右键菜单支持重命名、删除、新建子章节

### 三、角色管理

角色面板帮助你管理小说中的所有角色。

- **角色档案**：为每个角色记录姓名、别名（用于编辑器中的名称识别）、外貌描写、性格特点、背景故事
- **自定义属性**：可以添加任意自定义字段（如「武器」「能力」「阵营」等）
- **时间线快照**：角色在故事不同阶段可能有不同的外貌或性格变化，通过时间线快照记录这些变化
- **角色搜索**：支持按名称和别名快速搜索

### 四、角色关系

角色关系系统帮助你梳理复杂的人物关系网。

- **8 种关系类型**：亲属、朋友、敌人、师徒、恋人、盟友、上下级、自定义
- **关系强度**：每段关系可设置强度值，反映关系的亲密或紧张程度
- **关系图谱**：切换到关系图谱视图，以可视化网络图展示所有角色之间的关系
- **时间线关联**：关系可以绑定到特定时间线节点，追踪关系随剧情的变化

### 五、世界观构建

世界观面板帮助你构建和管理小说的背景设定。

- **11 种内置分类**：
  - 🏔️ 地点 — 城市、村庄、秘境等
  - ⚔️ 势力 — 门派、国家、组织等
  - 📜 规则 — 世界运行的法则
  - 🗡️ 物品 — 武器、法宝、道具等
  - 👥 种族 — 人族、妖族、精灵等
  - ✨ 魔法 — 法术体系、功法等
  - 📖 历史 — 重大历史事件
  - 🎭 文化 — 风俗、礼仪、信仰等
  - ⚙️ 科技 — 科技水平、发明等
  - 💰 经济 — 货币、贸易体系等
  - 🙏 宗教 — 神灵、教派等

- **自定义分类**：内置分类不够用？可以创建自己的分类
- **角色关联**：世界观词条可以关联到相关角色

### 六、时间线管理

时间线帮助你理清故事的时间脉络。

- **时间节点**：创建关键时间节点（如「天宝元年」「安史之乱」），添加描述
- **章节关联**：将时间节点关联到对应的章节，明确「这一章发生在什么时候」
- **角色关联**：标记哪些角色参与了这个时间节点的事件
- **排序调整**：拖拽调整时间节点的先后顺序

### 七、情节追踪

情节面板帮助你追踪小说中的各条故事线。

- **情节线索**：为每条故事线创建一个线索卡片
- **三种状态**：
  - 🔵 待展开 — 已埋下伏笔但尚未展开
  - 🟡 进行中 — 正在推进的情节
  - 🟢 已解决 — 已经收束的情节
- **章节关联**：标记这条情节线在哪些章节中出现

### 八、AI 辅助写作

火龙果编辑器集成了 AI 辅助写作功能，支持任何 OpenAI 兼容的 API。

#### 配置 AI

编辑 `public/ai-config.json`，填入你的 API 信息：

```json
{
  "providers": [
    {
      "id": "default-provider",
      "name": "智谱 GLM",
      "apiKey": "你的 API Key",
      "modelName": "glm-4-flash",
      "apiEndpoint": "https://open.bigmodel.cn/api/paas/v4/chat/completions",
      "timeoutMs": 60000
    }
  ],
  "activeProviderId": "default-provider"
}
```

支持的 AI 服务：
- 智谱 GLM（推荐，中文效果好）
- DeepSeek
- OpenAI（GPT-4 等）
- Ollama（本地模型）
- 任何兼容 OpenAI Chat Completions API 的服务

#### 8 种写作技能

| 技能 | 图标 | 用途 |
|------|------|------|
| 续写 | ✍️ | 根据当前内容自然续写，保持文风一致 |
| 润色 | 💎 | 优化句式、增强描写、提升文学表现力 |
| 对话 | 💬 | 生成符合角色性格的高质量对话（可选择角色） |
| 场景 | 🏞️ | 写沉浸式的场景描写，融合世界观设定 |
| 扩写 | 📝 | 将简略内容扩写为丰满立体的段落 |
| 改写 | 🔄 | 用不同视角或叙事方式改写当前内容 |
| 冲突 | 🎭 | 设计戏剧冲突，推动剧情发展 |
| 内心 | 💭 | 为角色写深入的内心独白（可选择角色） |

#### 智能推荐

AI 面板会根据当前章节的上下文自动推荐最适合的技能：
- 章节字数较少时推荐「扩写」
- 检测到对话内容时推荐「对话」
- 有角色关联时推荐「冲突」「内心」
- 有世界观设定时推荐「场景」

#### 上下文感知

AI 生成时会自动收集以下上下文信息，让生成内容更贴合你的故事：
- 当前章节的完整内容
- 前后章节的摘要
- 关联角色的信息（姓名、别名、外貌、性格、背景）
- 世界观设定
- 时间线上下文

#### 流式输出与取消

- AI 生成的内容以流式方式实时显示，无需等待全部生成完毕
- 生成过程中可随时点击「取消」按钮中止
- 快速连续点击不同技能时，系统自动取消上一个请求，只显示最新结果

#### 自定义技能

除了 8 种内置技能，你还可以创建自定义技能。技能使用 Markdown + YAML frontmatter 格式定义，支持参数化 prompt 和上下文推荐条件。

### 九、一致性检查

一致性检查功能帮助你发现章节中的角色名拼写错误。

- 系统基于编辑距离算法，自动检测与已定义角色名相似但不完全匹配的文本
- 对每个疑似错误提供修正建议
- 点击「应用」即可一键修正

### 十、导出功能

写完后，你可以将小说导出为多种格式：

| 格式 | 说明 |
|------|------|
| PDF | 适合打印和阅读，支持自定义字体、字号、行高、页边距 |
| EPUB | 电子书格式，可在 Kindle、iBooks 等阅读器中打开 |
| Markdown | 纯文本格式，方便在其他编辑器中继续编辑 |
| TXT | 按章节导出为独立 TXT 文件，ZIP 打包下载 |

### 十一、版本快照

版本快照功能让你可以随时回到之前的状态。

- **创建快照**：在重大修改前手动创建快照，附带备注说明
- **查看历史**：右侧面板显示所有快照，包含时间、备注和总字数
- **恢复快照**：选择任意快照恢复，系统会自动备份当前状态（防止误操作）
- **删除快照**：不需要的快照可以删除释放空间

### 十二、主题切换

- 点击工具栏的主题按钮切换亮色/暗色模式
- 支持「跟随系统」选项，自动适配操作系统的主题偏好
- 暗色模式适合夜间长时间写作，减少眼睛疲劳

## 数据存储说明

| 数据 | 存储位置 | 说明 |
|------|----------|------|
| 项目数据 | 本地 `.novel` 文件 | 包含所有章节、角色、世界观等数据 |
| 最近项目列表 | localStorage | 最多记录 10 个最近打开的项目 |
| 版本快照 | localStorage | 每个项目最多 50 个快照 |
| AI 生成历史 | localStorage | 每个项目最多 50 条记录 |
| 主题偏好 | localStorage | 亮色/暗色/跟随系统 |
| 日更目标 | localStorage | 目标字数和每日进度 |

所有数据均存储在你的本地电脑上，不会上传到任何服务器。
