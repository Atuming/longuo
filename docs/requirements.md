# 火龙果编辑器 — 需求文档

## 1. 项目概述

火龙果编辑器（Pitaya Editor）是一款面向中文小说作者的 Web 端写作助手工具。产品目标是为作者提供从创作到导出的一站式写作体验，集成 AI 辅助写作、角色管理、世界观构建、时间线管理、情节追踪、一致性检查和多格式导出等功能。

### 1.1 目标用户

- 网络小说作者（长篇连载）
- 轻小说 / 类型小说创作者
- 需要管理复杂角色关系和世界观的写作者
- 希望借助 AI 提升写作效率的创作者

### 1.2 核心价值

- 零安装、浏览器直接使用，数据存储在本地 `.novel` 文件中
- AI 辅助写作降低创作门槛，提供续写、润色、对话生成等 8 种写作技能
- 结构化管理角色、世界观、时间线、情节线索，避免长篇创作中的设定遗忘和矛盾
- 支持 PDF、EPUB、Markdown、TXT 多格式导出

## 2. 功能需求

### 2.1 项目管理

| 编号 | 需求 | 描述 |
|------|------|------|
| PM-1 | 创建项目 | 用户可创建新的小说项目，输入项目名称和描述，生成 `.novel` 文件 |
| PM-2 | 打开项目 | 用户可通过文件选择器打开已有的 `.novel` 文件 |
| PM-3 | 保存项目 | 用户可将当前项目数据保存到 `.novel` 文件，支持自动保存提示 |
| PM-4 | 最近项目 | 系统记录最近打开的项目（最多 10 个），支持快速重新打开 |
| PM-5 | 项目元数据 | 每个项目包含名称、描述、创建时间、更新时间等元数据 |

### 2.2 章节管理

| 编号 | 需求 | 描述 |
|------|------|------|
| CH-1 | 层级结构 | 支持卷（volume）→ 章（chapter）→ 节（section）三级层级 |
| CH-2 | 创建章节 | 用户可在任意层级下创建新章节 |
| CH-3 | 编辑章节 | 用户可编辑章节标题和内容 |
| CH-4 | 删除章节 | 删除章节时级联删除所有子章节 |
| CH-5 | 拖拽排序 | 用户可通过拖拽调整章节顺序和层级关系 |
| CH-6 | 字数统计 | 自动统计每个章节的字数（中文字符 + 英文单词），去除 Markdown 标记 |

### 2.3 写作编辑器

| 编号 | 需求 | 描述 |
|------|------|------|
| ED-1 | Markdown 编辑 | 基于 CodeMirror 6 的 Markdown 编辑器，支持语法高亮 |
| ED-2 | 角色名高亮 | 编辑器中自动高亮已定义的角色名称 |
| ED-3 | @提及补全 | 输入 `@` 时弹出角色名自动补全列表 |
| ED-4 | 悬浮提示 | 鼠标悬停角色名时显示角色详细信息 |
| ED-5 | 专注模式 | 隐藏侧边栏和面板，提供无干扰写作环境 |
| ED-6 | 日更目标 | 用户可设置每日写作字数目标，实时显示进度 |

### 2.4 角色管理

| 编号 | 需求 | 描述 |
|------|------|------|
| CR-1 | 角色档案 | 每个角色包含姓名、别名、外貌、性格、背景故事、自定义属性 |
| CR-2 | 角色 CRUD | 支持创建、查看、编辑、删除角色 |
| CR-3 | 角色搜索 | 支持按名称和别名搜索角色 |
| CR-4 | 时间线快照 | 角色在不同时间线节点可有不同的外貌和性格描述 |
| CR-5 | 角色关系 | 支持 8 种关系类型：亲属、朋友、敌人、师徒、恋人、盟友、上下级、自定义 |
| CR-6 | 关系图谱 | 可视化展示角色之间的关系网络 |
| CR-7 | 关系强度 | 每段关系可设置强度值 |

### 2.5 世界观管理

| 编号 | 需求 | 描述 |
|------|------|------|
| WD-1 | 内置分类 | 提供 11 种内置分类：地点、势力、规则、物品、种族、魔法、历史、文化、科技、经济、宗教 |
| WD-2 | 自定义分类 | 用户可创建自定义世界观分类 |
| WD-3 | 词条管理 | 每个词条包含名称、描述、分类、关联角色 |
| WD-4 | 分类筛选 | 支持按分类筛选世界观词条 |
| WD-5 | 词条搜索 | 支持按名称搜索世界观词条 |

### 2.6 时间线管理

| 编号 | 需求 | 描述 |
|------|------|------|
| TL-1 | 时间节点 | 创建时间线节点，包含标签、描述、排序 |
| TL-2 | 章节关联 | 时间节点可关联到具体章节 |
| TL-3 | 角色关联 | 时间节点可关联到具体角色 |
| TL-4 | 排序调整 | 支持拖拽调整时间节点顺序 |
| TL-5 | 级联删除 | 删除时间节点时级联删除关联的角色快照和关系记录 |

### 2.7 情节管理

| 编号 | 需求 | 描述 |
|------|------|------|
| PL-1 | 情节线索 | 创建情节线索，包含标题、描述、状态 |
| PL-2 | 状态追踪 | 三种状态：待展开、进行中、已解决 |
| PL-3 | 章节关联 | 情节线索可关联到具体章节 |
| PL-4 | 状态筛选 | 支持按状态筛选情节线索 |

### 2.8 AI 辅助写作

| 编号 | 需求 | 描述 |
|------|------|------|
| AI-1 | 写作技能 | 提供 8 种内置写作技能：续写、润色、对话、场景、扩写、改写、冲突、内心独白 |
| AI-2 | 技能参数 | 部分技能支持参数输入（如选择对话角色） |
| AI-3 | 上下文打包 | 自动收集当前章节内容、前后章节摘要、角色信息、世界观设定、时间线上下文 |
| AI-4 | 智能推荐 | 根据当前章节上下文信号（字数、对话、角色、世界观）推荐最适合的写作技能 |
| AI-5 | 流式输出 | AI 生成内容以流式方式实时显示 |
| AI-6 | 取消生成 | 用户可随时取消正在进行的 AI 生成请求 |
| AI-7 | 并发安全 | 新请求自动取消旧请求，防止内容混乱 |
| AI-8 | 生成历史 | 记录 AI 生成历史，支持查看和重新使用 |
| AI-9 | 多模型支持 | 支持任何 OpenAI 兼容 API（智谱 GLM、DeepSeek、Ollama 等） |
| AI-10 | 自定义技能 | 支持通过 Markdown+YAML 格式定义自定义写作技能 |
| AI-11 | 技能导入导出 | 支持以 zip 格式导入导出技能包 |

### 2.9 一致性检查

| 编号 | 需求 | 描述 |
|------|------|------|
| CC-1 | 角色名检测 | 基于编辑距离检测章节中可能的角色名拼写错误 |
| CC-2 | 修正建议 | 对检测到的问题提供修正建议 |
| CC-3 | 一键修正 | 用户可一键应用修正建议 |

### 2.10 导出功能

| 编号 | 需求 | 描述 |
|------|------|------|
| EX-1 | PDF 导出 | 支持导出为 PDF 格式，可自定义排版参数 |
| EX-2 | EPUB 导出 | 支持导出为 EPUB3 格式，包含完整元数据 |
| EX-3 | Markdown 导出 | 支持导出为完整 Markdown 文件 |
| EX-4 | TXT 导出 | 支持按章节导出为 TXT 文件（ZIP 打包） |
| EX-5 | 排版设置 | 可自定义字体、字号、行高、页边距 |

### 2.11 版本管理

| 编号 | 需求 | 描述 |
|------|------|------|
| VH-1 | 创建快照 | 用户可手动创建项目快照，附带备注 |
| VH-2 | 查看快照 | 查看快照列表，显示时间、备注、总字数 |
| VH-3 | 恢复快照 | 恢复到指定快照，恢复前自动备份当前状态 |
| VH-4 | 删除快照 | 支持删除不需要的快照 |

### 2.12 主题系统

| 编号 | 需求 | 描述 |
|------|------|------|
| TH-1 | 亮色模式 | 默认亮色主题 |
| TH-2 | 暗色模式 | 暗色主题，适合夜间写作 |
| TH-3 | 跟随系统 | 自动跟随操作系统的主题偏好 |
| TH-4 | 偏好持久化 | 主题偏好保存到 localStorage |

## 3. 非功能需求

| 编号 | 需求 | 描述 |
|------|------|------|
| NF-1 | 纯前端运行 | 无需后端服务器，所有数据存储在本地 |
| NF-2 | 离线可用 | 除 AI 功能外，所有功能离线可用 |
| NF-3 | 浏览器兼容 | 支持 Chrome、Edge 等现代浏览器（需 File System Access API） |
| NF-4 | 响应速度 | 编辑器输入延迟 < 50ms，页面切换 < 200ms |
| NF-5 | 数据安全 | 项目数据仅存储在用户本地，不上传到任何服务器 |
| NF-6 | 代码分割 | 编辑器页面懒加载，首屏加载时间 < 2s |
| NF-7 | 测试覆盖 | 核心业务逻辑有单元测试和属性基测试覆盖 |
